# Сценарии расширения 
## Stage 0
### Описание
В системе существует одно племя(`Tribe`).
Племя содержит фиксированное число одинаковых сущностей. Определён модуль, который симулирует игровой процесс.

Симуляция пошаговая, выполняется `N` раз. На каждом шаге каждая сущность(`Unit`) в племени независимо выполняет одно действие `act()` - простой вывод в консоль.

**Поддержка расширения на данном этапе отсутствует**. Код написан так, будто дальнейших изменений не будет.

## Stage 1
### Основные изменения
В проект добавлены разные типы сущностей для племени: `Warrior` и `Worker`, которые выполняют разные действия(индивидуальный вывод в консоль)

**Для ООП подхода**, `Warrior` и `Worker` - наследники класса `Unit`, в котором метод `act()` теперь виртуальный. Классы-наследники переопределяют метод `act()`.

**Для ППП подхода**, `Warrior` и `Worker` - специализации структуры `Unit`, которая теперь является обобщением. Через параметрические функции классы-специализацию реализуют свои версии метода `act()`. Метод `act()` в классе-обощении имеет пустую реализацию, которую можно вызвать(отсутствует механизм абстрактных структур).

Симуляция происходит также пошагово. Добавлена случайная генерация типа сущности.

## Stage 2
### Основные изменения
1. Два племени вместо одного, состав которых симметричен;
2. Для каждого `Unit` теперь есть значение здоровья(`health`);
3. Поведение разных типов `Unit` теперь сильно различается: 
    1. `Warrior` на каждом ходу атакует случайную живую единицу из вражеского племени. Урон зависит от типа цели: `Warrior` vs `Warrior` — урон уменьшается вдвое, `Warrior `vs `Worker` — полный урон.
    2. `Worker` на каждом ходу прибавляет случайное число ресурсов к счётчику своего племени.
4. На каждом шаге симуляции появляется параметр - контекст симуляции. Это структура, которая содежит для каждого племени своё и вражеское племя. Передавается каждому `Unit` при вызове `act()`.
