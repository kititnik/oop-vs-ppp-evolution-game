# Методология оценки качества архитектуры

### A. Статические характеристики

| Метрика | Значение |
|---|---|
| SLOC | Количество строк кода без пустых строк и без комментариев |
| Files | Число файлов |
| ΔSLOC% (Относительный объём кода PPP к OOP в процентах) | (SLOC_PPP − SLOC_OOP) / SLOC_OOP × 100% |

### B. Стоимость изменений

Измеряется при переходе от стадии N к N+1.

| Метрика | Значение |
|---|---|
| Net Growth | (+LOC) − (-LOC) |
| Modification Ratio | изменённые файлы / (изменённые + новые) | **Главный индикатор OCP**:  |

Modification Ratio - главный индикатор соблюдения OCP: 
- 0 = весь функционал добавлен через новые файлы
- 1 = весь функционал добавлен через изменения существующих файлов

### C. Характеристики расширяемости

| Метрика | Значение |
|---|---|
| Ce_avg | среднее число `#include` в каждом файле |
| Architectural Overhead Ratio | LOC инфраструктурного кода / Total SLOC |
| Architectural Overhead OOP/PPP | AO_OOP / AO_PPP |
| OCP Violation Points | число изменений для добавления нового типа Unit |

**Инфраструктурный код** - код, существующий из-за требований паттерна, а не из-за игровой логики.
